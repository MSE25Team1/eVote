1. Einführung in CI/CD – Aufgabenstellung

Was ist CI/CD? (in 2 Sätzen)

  • Continuous Integration (CI): Jede Codeänderung wird automatisch geprüft – Build, Tests, Qualitätssicherung.
  • Continuous Deployment (CD): Änderungen, die Tests bestehen, werden automatisch oder halbautomatisch auf einen Zielserver ausgeliefert.

CI und CD – Grundlagen

  • CI (Continuous Integration): Codeänderungen werden regelmäßig in das zentrale Repository integriert und automatisch getestet.
  • CD (Continuous Deployment/Delivery): Der geprüfte Code wird automatisch auf Test- oder Produktionsserver bereitgestellt.

Vorteile für das eVote-Projekt

  • Schnellere Entwicklung durch sofortige Prüfung und Integration von Änderungen.
  • Weniger Integrationsprobleme durch frühes Erkennen von Konflikten.
  • Stabile Releases dank automatisierter Tests und Builds.
  • Mehr Transparenz im Team durch sichtbaren Build-Status.

Automatische Tests & Builds

  • Automatische Builds: Prüfen, ob der Code kompiliert und lauffähig ist.
  • Unit- & Integrationstests: Erkennen Logikfehler direkt nach dem Commit.
  • Früherkennung von Fehlern: Probleme werden erkannt, bevor sie live gehen.
  • Zeitersparnis: Weniger manuelle Prüfungen – mehr Fokus auf neue Features.
